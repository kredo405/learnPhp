3.8. Добавление автоматических шаблонов в проект

В текущей статье мы попрактикуемся в работе со слотами в нашем проекте и добавим автоматические шаблоны (layouts) для страниц.

Для начала нам необходимо переработать AppLayout.vue компонент. Посмотрим на его изменения:

<template>
-  <div class="app-layout">
-    <AppLayoutHeader
-      :filters="filters"
-      @applyFilters="$emit('applyFilters', $event)"
-    />
-    <div class="content">
-      <AppLayoutMainSidebar
-        :tasks="tasks"
-        :filters="filters"
-        @updateTasks="$emit('updateTasks', $event)"
-      />
-      <IndexHome
-        :tasks="tasks"
-        :filters="filters"
-        @updateTasks="$emit('updateTasks', $event)"
-        @applyFilters="$emit('applyFilters', $event)"
-      />
-    </div>
-  </div>

+  <component
+    :is="layout"
+    :tasks="layoutTasks"
+    :filters="filters"
+    @updateTasks="$emit('updateTasks', $event)"
+    @applyFilters="$emit('applyFilters', $event)"
+  >
+    <slot />
+  </component>
</template>

<script>
- import AppLayoutMainSidebar from '@/layouts/AppLayoutMainSidebar';
- import AppLayoutHeader from '@/layouts/AppLayoutHeader';
- import IndexHome from '@/views/Index';

+ const defaultLayout = 'AppLayoutDefault';

  name: 'AppLayout',

    AppLayoutMainSidebar,

Сначала полностью уберём HTML-код из шаблона (template) и добавим <component>. Мы видим, что между тегами component (о нём мы поговорим ниже) мы добавили наш слот. В него будет помещён наш контент. Данный компонент является обёрткой для любого контента, который мы хотим поместить.

Тег component

Поговорим о теге component. Это не HTML-тег в прямом понимании этого слова. В мире Vue это называется динамический компонент. Что значит динамический? Это значит, что в зависимости от условий, которые определяет разработчик, динамический компонент будет преобразован в тот или иной компонент. Единственный обязательный атрибут динамического компонента — атрибут is. Он показывает динамическому компоненту, какой именно компонент нужно отрендерить.

Рассмотрим небольшой пример:

<template>
  <component :is="myComponent" />
</template>

<script>
import ComponentA from './ComponentA.vue'
import ComponentB from './ComponentB.vue'

export default {
  computed: {
    myComponent() {
      const date = new Date()
      const weekDay = d.getDay()
      return weekDay < 4 ? ComponentA : ComponentB
    }
  }
};
</script>

В этом примере: если день недели меньше 4 (четверг), то будет показан компонент А, в противном случае — компонент Б.

Продолжение работы с AppLayout.vue

Продолжим работу над нашим автоматическим шаблоном. Как видно из кода, добавленного в AppLayout.vue, мы внесли в наш компонент два computed-свойства:

computed: {
  layout() {
    const layout = this.$route.meta.layout || defaultLayout;
    return () => import(`@/layouts/${layout}.vue`);
  },

  layoutTasks() {
    return this.$route.meta.layout !== defaultLayout ? this.tasks : null;
  }
}

Наше свойство layout определяет, какой компонент следует отрендерить. Оно смотрит, есть ли у текущего маршрута (route) свойство layout и импортирует необходимый компонент для отображения. В случае если маршрут не обнаружен, то шаблон рендерится по умолчанию. Название шаблона по умолчанию мы вынесли в константу: const defaultLayout = 'AppLayoutDefault';.

Мы также убрали ненужные более импорты других компонентов и стили CSS.

Добавление шаблонов

Далее по списку нам необходимо добавить наши шаблоны: AppLayoutMain.vue и AppLayoutDefault.vue.

<template>
  <div class="app-layout-main">
    <AppLayoutHeader
      :filters="filters"
      @applyFilters="$emit('applyFilters', $event)"
    />
    <div class="content">
      <AppLayoutMainSidebar
        :tasks="tasks"
        :filters="filters"
        @updateTasks="$emit('updateTasks', $event)"
      />
      <slot />
    </div>
  </div>
</template>

<script>
import AppLayoutMainSidebar from '@/layouts/AppLayoutMainSidebar';
import AppLayoutHeader from '@/layouts/AppLayoutHeader';

export default {
  name: 'AppLayoutMain',
  components: {
    AppLayoutMainSidebar,
    AppLayoutHeader
  },
  props: {
    tasks: {
      type: Array,
      required: true
    },
    filters: {
      type: Object,
      required: true
    }
  }
};
</script>

Для добавления шаблона для маршрута нам нужно указать его наименование в свойстве meta.layout маршрута. Например:

{
  path: '/', // url
  name: 'IndexHome', // наименование маршрута
  component: () => import('../views/Index.vue'), // компонент для монтирования
  meta: {
    layout: 'AppLayoutMain' // применяемый шаблон
  },
}

Мой наставник

Для полноценного прохождения курса вам нужно выбрать наставника.

После выбора вы сможете всегда держать под рукой его контакты (почту и ссылки на профиль в соцсетях, которые он укажет для связи со своими студентами), а также расписание консультаций.

Вы в любой момент можете поменять наставника, если вам кажется, что с текущим наладить работу не получается.

Мои проекты

Для удобства работы с проектами, мы собрали их в этом блоке.

Основной личный проект, который вы будете готовить к финальной защите, находится в начале списка и рядом с ним стоит иконка . Если на курсе используется учебный проект, то он будет следующим и с иконкой  . Далее будут идти все остальные проекты, если для них были созданы репозитории.

Вы можете выбрать, какой именно проект будет основным, в любой момент прохождения курса, но мы советуем определиться с выбором как можно скорее.

Оцените наставника

Мы всегда стремимся улучшать качество услуг. И нам важно знать, как идёт ваша работа с наставником.

Уделите пару секунд вашего времени и дайте нам знать, как обстоят дела.

Также вы всегда можете задать свои вопросы напрямую куратору.

Ваши первые задачи

Выполняйте задачи по порядку, а система будет автоматически отмечать галочками ваш прогресс. Когда весь список будет выполнен, этот блок закроется и больше не будет вам надоедать.

Стандартный тариф

Студенты стандартного тарифа работают с наставником в течение первых пяти недель обучения. Оставшееся время консультаций, можно потратить на подготовку проекта к защите.

Запись на консультацию

Голосовая консультация — это живое общение с наставником, который ответит на интересующие вас вопросы.

Текстовая консультация — это общение с наставником в чате. Чат доступен 1 день с момента начала консультации.

Кодревью — это проверка пулреквеста в репозитории вашего проекта. Наставник оставит комментарии к вашему коду.

От выбранного раздела зависит, кто из наставников сможет провести консультацию.

Вы можете отправить заявку на консультацию конкретному наставнику, или её возьмёт первый, освободившийся из живой очереди.
