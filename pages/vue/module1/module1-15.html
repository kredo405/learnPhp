1.15. Практика по основам

Пришло время поработать над проектом. Вот наш план действий:

  1. Добавляем JSON тестовые данные.
  2. Создаём компонент хедера.
  3. Создаём компонент сайдбара.
  4. Создаём компонент доски задач.
  5. Объединяем компоненты в 1 лейауте.
  6. Подключаем лейаут в App.vue.

Важно: в теоретических статьях мы будем вырезать стили scss и заменять их на «...». Кроме того, части кода мы будем оформлять в виде комментариев, чтобы у студента было понимание, какая часть кода комментируется. Для облегчения восприятия материалов некоторые части кода могут не входить в статьи.

Окончательный вариант состояния проекта студент может посмотреть в репозитории.

Sidebar

Реализован в качестве части лейаута frontend/src/layouts/AppLayoutMainSidebar.vue.

В шаблоне мы используем v-for — для отрисовки списка задач и тегов внутри.

Используем v-if для статусов по важности и по времени, для блока тегов.

Используем интерполяцию для вывода количества задач, имени пользователя и названия задачи.

Используем динамические классы для статусов.

<script>
// Список задач, из которого достанем задачи для сайдбара.
import tasks from '@/static/tasks.json';
// Вспомогательные функции для нормализации задач и тегов задач.
import { normalizeTask, getTagsArrayFromString } from '@/common/helpers';

export default {
  name: 'AppLayoutMainSidebar',
  data() {
    return {
      // получаем нормализованные задачи
      tasks: tasks.map(task => normalizeTask(task))
      backlogIsHidden: false
    };
  },
  computed: {
    // Выбираем только задачи без id колонки (у нас это именно задачи сайдбара).
    // Нормализуем список тегов.
    // Сортируем задачи по их порядку внутри колонки.
    sidebarTasks() {
      return this.tasks
        .filter(task => !task.columnId)
        .map(task => ({
          ...task,
          tags: getTagsArrayFromString(task.tags)
        }));
    }
  }
};
</script>

Вспомогательные методы и константы

Список задач в том виде, в котором мы будем получать его с сервера, отличается от формата, который мы будем использовать на фронтенде. Чтобы решить эту задачу, мы сделали вспомогательные методы и константы. Файл с методами находится в frontend/src/common/helpers.js.

import {
  DAY_IN_MILLISEC,
  TAG_SEPARATOR
} from '@/common/constants';
import timeStatuses from '@/common/enums/timeStatuses';
import taskStatuses from '@/common/enums/taskStatuses';

// Получение тегов из строки тегов.
export const getTagsArrayFromString = tags => {
  const array = tags.split(TAG_SEPARATOR);
  return array.slice(1, array.length);
};

// Нормализация статуса по времени.
export const getTimeStatus = dueDate => {
  if (!dueDate) {
    return '';
  }
  const currentTime = +new Date();
  const taskTime = Date.parse(dueDate);
  const timeDelta = taskTime - currentTime;
  if (timeDelta > DAY_IN_MILLISEC) {
    return '';
  }
  return timeDelta < 0 ? timeStatuses.deadline : timeStatuses.expired;
};

// Нормализация задачи.
export const normalizeTask = task => {
  return {
    ...task,
    status: task.statusId ? taskStatuses[task.statusId] : '',
    timeStatus: getTimeStatus(task.dueDate)
  };
};

<script>
// Импортируем JSON данные и статусы для фильтров.
import columns from '@/static/columns.json';
import tasks from '@/static/tasks.json';
import users from '@/static/users.json';
import { statuses } from '@/common/constants';
import {
  getTagsArrayFromString,
  normalizeTask
} from '@/common/helpers';

export default {
  name: 'IndexHome',
  data() {
    return {
      columns,
      tasks: tasks.map(task => normalizeTask(task)),
      users,
      statuses,
      taskStatuses
    };
  },
  computed: {
    // Получаем список задач для колонок в формате объекта
    // (ключ - id колонки, значение - массив задач колонки)
    columnTasks() {
      return this.tasks
        .filter(({ columnId }) => columnId)
        .reduce((accumulator, task) => {
          task.tags = getTagsArrayFromString(task.tags);
          if (accumulator[task.columnId]) {
            accumulator[task.columnId] = [...accumulator[task.columnId], task];
          } else {
            accumulator[task.columnId] = [task];
          }
          return accumulator;
        }, {});
    }
  }
};

Мой наставник

Для полноценного прохождения курса вам нужно выбрать наставника.

После выбора вы сможете всегда держать под рукой его контакты (почту и ссылки на профиль в соцсетях, которые он укажет для связи со своими студентами), а также расписание консультаций.

Вы в любой момент можете поменять наставника, если вам кажется, что с текущим наладить работу не получается.

Мои проекты

Для удобства работы с проектами, мы собрали их в этом блоке.

Основной личный проект, который вы будете готовить к финальной защите, находится в начале списка и рядом с ним стоит иконка . Если на курсе используется учебный проект, то он будет следующим и с иконкой  . Далее будут идти все остальные проекты, если для них были созданы репозитории.

Вы можете выбрать, какой именно проект будет основным, в любой момент прохождения курса, но мы советуем определиться с выбором как можно скорее.

Стандартный тариф

Студенты стандартного тарифа работают с наставником в течение первых пяти недель обучения. Оставшееся время консультаций, можно потратить на подготовку проекта к защите.

Запись на консультацию

Голосовая консультация — это живое общение с наставником, который ответит на интересующие вас вопросы.

Текстовая консультация — это общение с наставником в чате. Чат доступен 1 день с момента начала консультации.

Кодревью — это проверка пулреквеста в репозитории вашего проекта. Наставник оставит комментарии к вашему коду.

От выбранного раздела зависит, кто из наставников сможет провести консультацию.

Вы можете отправить заявку на консультацию конкретному наставнику, или её возьмёт первый, освободившийся из живой очереди.
