1.6. Компонент: шаблон (template)

Шаблон Vue.js — это валидная HTML-разметка, внутри которой могут использоваться свойства экземпляра Vue. Свойства определяются в блоке script. Для работы Vue компилирует внутри шаблоны в render-функции виртуального DOM.

Давайте разберём, какие особенности есть у шаблонов:

  1. Интерполяции:
    • Текст.
    • Сырой HTML.
    • Атрибуты.
    • Выражения JavaScript.
  2. Директивы:
    • Аргументы.
    • Динамические аргументы.
    • Модификаторы.
  3. Сокращения:
    • Сокращение v-bind.
    • Сокращение v-on.

Шаблон является частью экземпляра Vue. Ему доступны свойства из блока скриптов этого же экземпляра. Интерполяция значит, что внутри строки шаблона можно вставить свойство экземпляра, и оно будет преобразовано в значение.

Интерполяция: текст

Текстовая интерполяция в Vue-шаблоне реализована с использованием синтаксиса Mustache (двойных фигурных скобок). Такую интерполяцию мы будем использовать очень часто. Выглядит она вот так:

<span class=”username”>{{ username }}</span>

Выражение в фигурных скобках будет заменено на значение свойства username. И что очень важно, в DOM значение будет обновлено при каждом изменении значения username.

Интерполяция: сырой HTML

Если мы будем использовать синтаксис с фигурными скобками и передадим в качестве значения HTML-строку, она будет отображена как текст. Если же мы хотим трактовать строку именно как HTML-вставку, для этого существует директива v-html.

// htmlString = `<span style=”color:red;”>Красный текст</span>`

// Вместо htmlString код выведется так, как мы его написали выше в переменной
<p>{{ htmlString }}</p>

// В этом случае выведется текст: "Красный текст" красного цвета
<p><span v-html="htmlString"></span></p>

Динамическая отрисовка произвольного HTML-кода крайне опасна, так как может привести к XSS-уязвимостям. Используйте интерполяцию HTML только для доверенного кода и никогда не подставляйте туда содержимое, создаваемое пользователями.

На практике HTML-вставки используются намного реже, чем текстовые интерполяции и интерполяции атрибутов.

Интерполяция: атрибуты

Для интерполяции атрибутов используется синтаксис v-bind.

<button v-bind:style="btnStyle">Кнопка</button>
<button v-bind:disabled="isDisabled">Кнопка</button>

При использовании с булевыми атрибутами если значением isDisabled будет null, undefined или false, то атрибут disabled не добавится в элемент <button>.

В примере мы указали style и disabled для простоты понимания. На самом деле, вы можете сделать динамическим любой атрибут внутри html-тега в шаблоне.

Интерполяция: выражения JavaScript

{{ num + 1 }}
{{ isLoading ? 'Loading...' : 'Is loaded' }}
{{ message.split('').reverse().join('') }}
<div v-bind:id="'item-' + id"></div>

Выражения будут вычислены как JavaScript-код в области видимости соответствующего экземпляра Vue. Единственное ограничение в том, что допускается лишь одно выражение, поэтому код ниже НЕ сработает:

<!-- это не вычисляемое выражение, а определение переменной: -->
{{ var a = 1 }}

<!-- операторы условий не сработают, используйте условные операторы в тернарной форме: -->
{{ if (ok) { return message } }}

Выражения в шаблонах выполняются в режиме «песочницы», поэтому доступ есть только к ограниченному списку глобальных объектов, таким как Math и Date. Не пытайтесь получить доступ к пользовательским глобальным объектам внутри выражений, используемых в шаблонах.

Директивы

Директивы — это специальные атрибуты с префиксом v-. В качестве значения они принимают одно выражение JavaScript (за исключением v-for). Наиболее часто используемые директивы:

v-bind (сокращается как ': '):

Создаёт динамические атрибуты. Привязывает значение свойства к HTML-атрибуту в тегах в шаблоне (class, style, id, href и т. д.). Например: v-bind:href="itemUrl", v-bind:id="itemId". Значения привязываются реактивно. При изменении значения свойства изменится и значение атрибута.

v-on (сокращается как '@'): Создаёт обработчик на событие (event). Поддерживаются все стандартные события DOM. Название события передаётся через двоеточие. Например: v-on:click="getData". Через точку после названия события можно добавить модификатор. Модификаторы можно объединять в цепочки (использовать сразу несколько). Например: v-on:click.stop.prevent="submitForm".

v-model: Создаёт двустороннюю привязку HTML-элемента и свойства. Это очень удобно для работы с полями форм.

То есть мы можем сделать ввод в input, и изменится привязанное свойство, и наоборот — изменить свойство и значение input так же изменится. Внутри v-model реализован как v-bind, так и v-on. Иногда бывает удобнее использовать их отдельно вместо v-model.

v-show: Задаёт условие, при котором необходимо показывать/скрывать элемент. Элемент с v-show будет всегда оставаться в DOM, а меняться будет лишь свойство display в его параметрах CSS.

v-show не работает на элементе <template> и не работает с v-else.

v-if: Задаёт условие, при котором необходимо показывать/скрывать элемент. Отличается от v-show тем, что в значении false элемент не добавляется в DOM-дерево, и добавляется, только когда значение — true.

v-else: Дополнение к v-if. Работает как if/else в чистом JavaScript. Тегу, следующему сразу после тега с v-if, добавляем v-else без параметров.

v-else-if: На практике используется реже, чем v-if и v-else. Логика работы такая же, как и else if в JavaScript.

v-for: Создан для отрисовки списка элементов. Можно применять для массивов и для объектов. Разберём детальнее в следующем разделе.

v-once: Отрисовывает элемент однократно, и при дальнейших обновлениях не реагирует на изменения.

v-html: Отрисовка переданной строки как HTML (говорили о нём выше).

Атрибуты

Некоторые директивы могут принимать «аргумент», отделённый от названия директивы двоеточием. Например:

<a v-bind:href="url"> ... </a>

href — аргумент, указывающий директиве v-bind связать атрибут href элемента со значением выражения url.

Динамические атрибуты

<a v-bind:[attributeName]="attributeValue"> ... </a>
<a v-on:[eventName]="eventHandler"> ... </a>

Таким образом, мы можем создавать динамические обработчики событий и атрибуты. На практике используются гораздо реже, нежели обычные атрибуты.

У динамического аргумента есть ограничения. Его значение всегда должно быть строчной строкой без пробелов. Также допускается значение null — так мы явно указываем, что в этом случае привязки атрибута не будет.

Модификаторы

Модификаторы — это постфиксы, добавляемые после точки. Они делают директиву более специфической. Пример модификатора: v-on:submit.prevent. Модификатор .prevent говорит директиве v-on вызвать event.preventDefault() при обработке произошедшего события.

Сокращения

Директивы v-bind и v-on можно сокращать. Например, v-bind:id можно записать как :id. То есть v-bind отбрасывается и остаётся только двоеточие и название атрибута.

v-on:click можно записать как @click. То есть v-on отбрасывается и вместо двоеточия используется символ @.

В проектах мы используем сокращённый синтаксис.

Стандартный тариф

Студенты стандартного тарифа работают с наставником в течение первых пяти недель обучения. Оставшееся время консультаций, можно потратить на подготовку проекта к защите.

Запись на консультацию

Голосовая консультация — это живое общение с наставником, который ответит на интересующие вас вопросы.

Текстовая консультация — это общение с наставником в чате. Чат доступен 1 день с момента начала консультации.

Кодревью — это проверка пулреквеста в репозитории вашего проекта. Наставник оставит комментарии к вашему коду.

От выбранного раздела зависит, кто из наставников сможет провести консультацию.
