Базовая теория

Глава 6.6. Родительский класс объектов Object3D

Вернёмся к объектам в Three.js. Все объекты наследуются от Object3D и могут вкладываться друг в друга, а именно помещаться в СК родителей. Для того чтобы отобразиться на экране объекты или их родители должны быть помещены на Scene.

Для чего нужна такая система? Для удобства. Нам на много проще сказать, что заднее колесо машины располагается на оси задней подвески, положение которой мы уже знаем, например, на расстоянии X от центра оси, чем в каждый момент времени описывать точное положение колеса в пространстве независимо от машины и оси.

И таким образом мы описываем положение всех деталей. Мы можем использовать любое количество уровней вложенности.

Что происходит при рендеринге?

Three.js, имея в распоряжении всё дерево зависимостей собирает данные о положении финальных дочерних элементов — не пустых, то есть имеющих инструкции для отображения: геометрии и материалы.

Положение центра одного Object3D задается следующим образом: 

object3d.position.set(0, 0, 0);	// -> Vector3(0, 0, 0) или new Vector3(), 0 - значения по умолчанию для координат

Размеры относительно начально описанных геометрией задаются свойством scale:

object3d.scale.set(1, 1, 1); // -> Vector3(1, 1, 1);

Scale по разным осям можно задавать отдельно.

Поворот Object3D можно задавать двумя способами.

Более привычный для нас в градусах. В 3D такой способ называется углами Эйлера: сначала мы поворачиваем объект относительно одной оси на определенное количество градусов, потом уже повернутый объект относительно другой оси и, наконец, третьей. Результирующее положение будет зависеть от порядка, и в котором мы будем применять повороты. 

object3d.rotation = new Euler(0, 1, 1.57, 'XYZ');

// то же, что и

object3d.rotation = new Euler(0, 1, 1.57);

// так как порядок 'XYZ' - по умолчанию

Второй способ — кватернионы (Quaternion). О том что такое кватернион.

Подробнее о классе Quaternion.

Оперировать вручную кватернионами неудобно, они далеки от интуитивно понятных углов. Однако они решают некоторые технические сложности с углами Эйлера и используются внутри Three.js для их решения. На данном этапе вам нет необходимости разбираться в этой теме.

Векторы и операции над векторами

Vector{2, 3, 4} — JS-классы для представления векторов 2, 3, и 4 размерностей. Аналоги vec{2, 3, 4} в GLSL.

В отличие от векторов в GLSL операции между векторами в Three.js осуществляются с помощью методов класса. Например, метод add складывает векторы, при этом модифицируя тот вектор, чей метод вызывается: 

const vector0 = new Vector3(1, 0, 0);
const vector1 = new Vector3(0, 1, 0);

vector0.add(vector1);		// -> Vector3(1, 1, 0)

Векторы можно вращать с помощью углов Эйлера или кватернионов:

const vector = new Vector3(1, 2, 0);

const euler = new Euler(0, 1, 1.57, 'XYZ')

const quaternion = new THREE.Quaternion();
quaternion.setFromAxisAngle( new THREE.Vector3( 0, 1, 0 ), Math.PI / 2 );

vector.applyEuler(euler);
vector.applyQuaternion(quaternion);

Для работы с векторами также имеются матрицы Matrix{3, 4} — 3×3 и 4×4 соответственно.

Изучите доступные методы векторов для операций над ними, в том числе с использованием матриц здесь.

Матрицы трансформаций

Три составляющие пространственного описания Object3D object3d.position, object3d.scale, object3d.rotation описывают трансформации СК, в которой размещается объект. А мы знаем, что трансформации переноса, вращения и размера могут быть описаны в одной матрице.

Чтобы получить такую матрицу из object3d.position, object3d.scale, object3d.rotation имеются методы .setPosition, .scale, .makeRotationFromEuler и другие методы для частных видов операций.

Далее к матрице объекта применяется матрица трансформации родителя и так до Scene. Результирующая матрица будет описывать положение относительно Scene или, как говорят, относительно глобальной или мировой СК matrixWorld.

После этого на матрицу в мировой СК воздействуют таким образом, чтобы узнать, в каком положении находится объект относительно камеры. Ведь именно на плоскость камеры нам нужно сделать проекцию в вершинном шейдере.

Таким образом с Object3D ассоциировано три матрицы:

  • Object3D.matrix — локальная матрица относительно родителя.
  • Object3D.matrixWorld — относительно Scene, мировой СК.
  • Object3D.modelViewMatrix — относительно положения Camera.

Последняя передается в вершинный шейдер и используется для расчета положения проекции вершины на экране.

Camera является также представителем Object3D. У Camera имеется две дополнительные матрицы:

  • Camera.matrixWorldInverse — матрица обратная к.matrixWorld, используется для расчета.modelViewMatrix других объектов относительно этой камеры.
  • Camera.projectionMatrix — содержит информацию о том, как проецировать сцену, чтобы обрезать пространство.

Код минимального вершинного шейдера в Three.js

// Переменные, которые передаёт Three.js для проецирования на плоскость
uniform mat4 projectionMatrix;
uniform mat4 modelMatrix;
uniform mat4 viewMatrix;

// Атрибуты вершины из геометрии
attribute vec3 position;
attribute vec3 normal;
attribute vec2 uv;

// Varying-переменная для передачи uv во фрагментный шейдер
varying vec2 vUv;

void main() {

	vUv = uv;

	gl_Position = projectionMatrix * viewMatrix * modelMatrix * vec4( position, 1.0 );

}

Названия матриц в JS и в GLSL не совпадают: в modelMatrix передаётся Object3D.matrixWorld, в viewMatrix — Camera.matrixWorldInverse, а в projectionMatrix — Camera.projectionMatrix.

Помните, что операция умножения матриц и векторов не является коммутативной, то есть нельзя поменять матрицу и вектор местами, а перемена мест матриц приведет к изменению результата.
