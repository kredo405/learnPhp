Базовая теория

Глава 5.8. Синтаксис GLSL

GLSL или Graphics Library Shader Language — язык программирования шейдеров графической библиотеки.

GLSL базируется на языке программирования ANSI C и является его упрощённой версией в связи со спецификой задач, для которых GLSL разработан — для растеризации графики.

В GLSL добавлены типы данных, операторы и функции, необходимые для работы с векторной графикой в вершинном шейдере и с растровой во фрагментном, в основном для работы с векторами разных размерностей: vec2, vec3, vec4 — двухмерными, трёхмерными и четырёхмерными соответственно.

Для разных действий над векторами существуют типы данных матриц: mat2, mat3 и mat4 для матриц 2×2, 3×3 и 4×4.

Матрицы — это массивы данных, описывающие изменение вектора: перемещение, поворот и изменение размера.

Существует математический аппарат для работы с векторами и матрицами. На текущем курсе рассмотрим лишь общие возможности векторов и матриц, необходимые для решения конкретных задач.

В начале раздела предлагаем почитать статьи на тему векторов и матриц, в частности матриц трансформации. Если вы уже знакомы с этим, статьи можно пропустить.

Тема работы с векторами и матрицами выходит далеко за пределы курса. Для лучшего понимания операций с ними, и чтобы полнее использовать возможности WebGL в будущем, можно изучить основы векторной алгебры и аналитической геометрии.

Теперь перейдём к синтаксису GLSL.

Объявление переменных

float a = 2.0;

Сперва идёт тип данных, через пробел — название переменной, через знак = значение.

Обязательно ставить ; в конце.

GLSL — строго типизированный язык. В отличие от JS, числа в GLSL различаются на int и float, и не преобразуются друг в друга автоматически.

То есть такой код вызовет ошибку:

float a = 2;

Чтобы исправить это, нужно либо обязательно указать .:

float a = 2.0;

либо привести тип с помощью функции:

float a = float(2);

Точность

Перед объявлением типа данных можно указать квалификатор точности, с которой будет считаться переменная:

highp float a = 2.0;
  • highp — наиболее высокая доступная точность расчётов чисел с плавающей точкой;
  • mediump — средняя точность между самой высокой и низкой;
  • lowp — низкая точность.

Точность может быть указана у отдельных переменных не только типа float, но и содержащих в составе float-числа.

Тип можно задать глобально в начале шейдера:

precision highp float;

Объявление точности у отдельных переменных переопределит точность данной переменной.

Квалификаторы переменных

Квалификаторы определяют смысл переменных в шейдерах.

В GLSL доступны вот такие квалификаторы переменных:

  • const — объявление константы;

  • attribute — глобальные переменные, которые могут меняться для каждой вершины, и передаются в вершинные шейдеры.

    Могут использоваться только в вершинных шейдерах. Для шейдера — read-only.

  • uniform — глобальная переменная, которая может меняться для каждого полигона, передаётся OpenGL в шейдеры.

    Может использоваться в обоих типах шейдеров. Для шейдеров — read-only.

  • varying используются для передачи интерполированных данных между вершинным и фрагментным шейдерами.

    Доступны для записи в вершинном шейдере, и read-only для фрагментного шейдера.

Функции

В GLSL есть перечень функций для работы с данными при растеризации. Их изучим в главе 5.11: «Функции в WebGL».

Вызов функции

Производится привычным способом:

normalize(v1);

Объявление функции

Разработчик может объявлять свои функции:

vec3 changeDirection(in vec3 dir, in mat4 matrix) {
	...
	return newVector;
}

Функция может возвращать значение как результат своей работы. В этом случае нужно указать тип возвращаемой величины. Если функция не возвращает ничего, нужно указать директиву void.

Параметры функции могут иметь следующие квалификаторы:

  • in — для входящих параметров (по умолчанию);
  • out — выходные данные функции;
  • inout — параметры, которые могут быть и входящими, и выходными.
void setWhite(out vec4 c)
{
    c = vec4(1.0);
}

void main() {
    vec4 color = vec4(0.0, 1.0, 0.0, 1.0);

    setWhite(out color);	// меняет значение цвета в переменной на белый

    gl_FragColor = color;
}

Квалификатор in указывать не обязательно.

Рекурсивное поведение функций GLSL не указано в спецификации, поэтому не нужно вызывать функции в её же корне.

Выражения

В GLSL доступно выражение if-else:

if(bool условие 1) ... 
else if(bool условие 2) ... 
else

Доступны «прыжковые» директивы:

  • continue;
  • break;
  • discard — используется только во фрагментных шейдерах, вызывает терминацию шейдера для текущего пикселя без передачи его далее во FrameBuffer или буфер глубины.

В спецификации описаны также привычные JS-разработчикам выражения for, while, do-while. Однако циклы не могут производиться параллельно, что вызывает торможение вычислений. К ним нужно прибегать лишь в крайних случаях. Например, их используют при Raycasting (метод определения объекта, на который указывает курсор).

Операции с векторами

GLSL разработан для работы с векторами и матрицами, и в отличие от JS даёт возможность производить манипуляции с ними не с помощью методов или функций, а используя привычные операторы сложения, вычитания и умножения.

Например, вектор можно умножить на скалярное значение: 

float a = 2.0;
vec4 v1 = vec4(1.0, 2.0, 3.0, 4.0);

vec4 v2 = v1 * a;  // ->  vec4(2.0, 4.0, 6.0, 8.0)

Аналогично можно умножать матрицы друг на друга и матрицы на векторы:

mat4 a = mat4(...);	// некая матрица 4x4
mat4 b = mat4(...);	// другая матрица 4x4
mat4 c = a * b;
 
vec4 v = vec4(1, 2, 3, 4);
vec4 y = c * v;		// -> трансформированный вектор

В vec4(1, 2, 3, 4) не указана точка для float значений, потому что vec2, vec3 и vec4 преобразуют целые числа в числа с плавающей точкой, как float(a).

Тем не менее лучше указывать точки в явном виде, так будет проще ориентироваться в коде и не делать синтаксических ошибок там, где точки обязательны.

Операции +, −, ***** и / могут производиться с двумя векторами и с вектором и скалярным числом. В этом случае операции будут выполняться по компонентам и записываться в новый вектор той же размерности.

Упрощённый доступ к компонентам векторов

В GLSL есть ряд конструкций, позволяющих получать разные компоненты векторов и преобразовывать их в другие векторы:

vec4 v = vec4(1, 2, 3, 4);
  • v.x то же, что и v.s или v.r или v[0];
  • v.y то же, что и v.t или v.g или v[1];
  • v.z то же, что и v.p или v.b или v[2];
  • v.w то же, что и v.q или v.a или v[3].

То, что цвет в WebGL представляется в форме вектора, обсудим в главе 5.19: «Цвет как вектор».

Координаты точек, например, вершин, и направленных векторов в пространстве задаются в vec4. Для чего же четвёртый параметр v.w? Он показывает, является ли vec4 точкой или направленным вектором: v.w = 1.0 у точки и v.w = 0.0 у направленного вектора, например, вектора скорости или смещения.

Также из значений вектора можно создавать новые векторы, просто перечислив названия компонент, значения которых нужны.

vec4 v = vec4(1, 2, 3, 4);

vec4 v1 = v.yyyy;			// -> vec4(2, 2, 2, 2)
vec4 v2 = vec4(v.y, v.y, v.y, v.y);	// -> vec4(2, 2, 2, 2)

vec3 v1 = v.bgr;			// -> vec3(3, 2, 1)
vec3 v1 = vec3(v.b, v.g, v.r); 	// -> vec3(3, 2, 1)

При создании vec или mat можно передать несколько компонентов зараз:

vec4(v.rgb, 1)

// делает то же, что и
vec4(v.r, v.g, v.b, 1)

или:

vec4(2)

// делает то же, что и
vec4(2, 2, 2, 2)

Ссылка на полную спецификацию GLSL есть в конце раздела в «Дополнительных материалах».

Как и для других языков во frontend, для GLSL есть препроцессоры, и можно работать с кодом в файлах с расширением *.glsl. В IDE такой код просвечивает и делается более читабельным. Во время сборки код минифицируется, чем уменьшает вес. Кроме этого, могут быть подставлены численные значения из специально заявленных констант, и это снижает нагрузку. Например:

@define PI 3.1415
// ....
float angle = 2.0 * PI;

Пример препроцессора: webpack-glsl-minify
