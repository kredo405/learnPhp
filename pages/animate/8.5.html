Базовая теория

Глава 8.5. Графические редакторы анимации для использования в web

Анимация — это трудоёмкий процесс и кропотливая работа. Создание анимации длинных сцен с большим количеством деталей может занимать месяц и даже дольше, если выполнять их вручную и тем более без использования библиотек.

Работа над крупными проектами может требовать большого опыта в анимации. Часто frontend-разработчики им не обладают, и это в порядке вещей. Нужно делегировать такие задачи опытным аниматорам.

Есть программы и онлайн-инструменты для анимации, которые можно использовать для набросков и прототипирования, или компилировать в код JS и JSON и использовать в web.

Инструменты для прототипирования

Эта категория программ используется не столько для полноценной детализированной работы над анимацией, сколько для наброска функционирования сайта или приложения и демонстрации переходов элементов между состояниями. Такие приложения отлично подходят для того, чтобы экономить время на этапе проектирования интерфейсов и подготовки дизайна для презентации клиенту.

Marvel

Работает из браузера или как дополнение к популярным графическим редакторам. Подходит для настройки переходов и позволяет просматривать прототип на смартфоне.

Сайт: marvelapp.com

Знакомство с Marvel

Цена: несколько тарифных планов, есть бесплатный.

Недостаток: требует интернет-соединения.

Principle

Подходит для несложной анимации и создания коротких сценариев в 2–4 экрана. Простой интерфейс. Есть возможность работать с таймлайном в виде диаграммы Ганта и настраивать easing.

Сайт: principleformac.com

Видеообзор Priciple

Цена: 129$ на год.

Недостаток: работает только под MacOS.

Конструкторы сайтов

Конструкторы позволяют быстро создавать сайты, пользуясь простым интерфейсом и не обладая глубокими знаниями в разработке. Они предоставляют ряд шаблонов для начала работы и обладают полезными функциями, например, оптимизацией под мобильные устройства, SEO, инструментами для электронной коммерции.

На 2020 год есть много разнообразных конструкторов. Некоторые из них имеют редакторы простой анимации и их можно использовать для создания лендингов или набросков и прототипов анимации для предстоящей разработки.

Например, конструктор Readymag позволяет создавать длинные одностраничные истории, настраивать состояния элементов и анимацию по скроллу. На бесплатном тарифе есть возможность работать над неограниченным количеством проектов, один из которых может быть опубликован в web.

Графические редакторы анимации

Графические редакторы позволяют полноценно работать с таймлайном, импортировать растровую и векторную графику, экспортировать результат и проигрывать на сайте с помощью JS.

Google Web Designer

Предназначен для создания анимированных html5-баннеров. Включает шаблоны баннеров для размещения в рекламной системе Google, а также имеет возможность настройки кастомных шаблонов. Включает готовые компоненты галерей, видеоплееров, карт. Позволяет работать не только с CSS 2D-трансформациями, но и с 3D. Есть возможность редактировать HTML, CSS и JS напрямую.

Сайт: Google Web Designer

Adobe Animate

Программа для создания сложной анимации в canvas:

  • базируется на работе в таймайне;
  • позволяет инкапсулировать анимацию в классах в библиотеке и использовать её в других классах как вложенные экземпляры MovieClip;
  • позволяет применять трансформации, морфинг, маскирование в анимации;
  • имеет редактор easing;
  • имеет возможность подключать шрифты и переводить текст в кривые;
  • позволяет оптимизировать загрузку изображений путём объединения их в атласы изображений.

Совместима с редакторами графики Adobe: импортирует графику из Adobe Photoshop и Adobe Illustrator в слои в таймлайне.

В результате экспортируется JS-библиотека, в которой содержатся все используемые в финальной сцене классы MovieClip (и другие служебные классы). Все классы этой библиотеки расширяют классы базовой, которую использует Adobe Animate: CreateJS.

CreateJS — результат переноса в JS языка разработки Action Script для Flash, от которого произошёл Adobe Animate. Action Script был объектно-ориентированным языком, и библиотека CreateJS также является объектно-ориентированной и воспроизводит объектную структуру Action Script и Adobe Animate.

CreateJS включает четыре составляющие:

  • EaselJS для рисования графики в canvas;
  • TweenJS — система для tween-анимации объектов из EaselJS;
  • SoundJS — библиотека для облегчения подключения аудиосопровождения;
  • PreloadJS для предварительной загрузки изображений и аудиофайлов.

MovieClip представляет собой контейнер, содержащий графику — изображения Bitmap, векторные формы Shape или дочерние MovieClip, разложенные по слоям. У каждого слоя в таймлайне могут быть отмечены ключевые фреймы для Tween или Morphing-анимации. У слоя также можно задать маску, вне которой изображение не будет рендериться.

MovieClip предоставляет интерфейс для добавления, удаления слоёв или изменения их порядка, а также для управления проигрыванием анимации. Фреймам можно задать имена и использовать их для управления анимацией. Например, можно вызвать movieclip.gotoAndPlay('myFabulousAnimationStart').

MovieClip для записи и воспроизведения анимации использует объект класса Tween из библиотеки TweenJS.

Для рендеринга MovieClip в canvas есть специальный класс stage = new createjs.Stage("demoCanvas"), имеющий ссылку на canvas. MovieClip или его родителя нужно добавить на Stage, чтобы он учитывался в рендеринге. Для этого нужно использовать метод stage.addChild(movieclip), а в каждом новом фрейме вызывать stage.update().

Кроме того, для анимации объектов CreateJS можно пользоваться и другими библиотеками, которые позволяют делать Tween абстрактным параметрам объектов. Например, библиотека GSAP может анимировать любые численные свойства абстракций. В этом случае можно использовать Adobe Animate для этапа подготовки графики и части анимации, а остальную работу проводить исключительно в коде JS.

Для более подробной информации ознакомьтесь с документацией: EaselJS, TweenJS.

Сайты: Adobe Animate, createjs.com

Adobe After Effects + Lottie

Adobe After Effects предназначен для работы с видео: composing и добавления растровых эффектов, графического оформления, маскирования и трекинга.

Создатели плагина Bodymovin и библиотеки Lottie использовали возможности Adobe After Effects по анимации векторной графики и анимации слоёв изображений для анимации в web и приложениях на мобильных устройствах.

Их плагин Bodymovin экспортирует анимацию в формате JSON. Затем библиотека lottie-web (для других платформ библиотеки, например, lottie-ios, lottie-android) воспроизводит эту анимацию в браузере.

При создании экземпляра анимации lottie-web можно указать один из способов рендеринга: svg, html или canvas (не рекомендуем использовать canvas, не нашлось примера его успешного использования, создатели главным образом ориентировались на векторную графику в svg).

Рендеринг анимации svg нужно использовать осторожно, анимация некоторых свойств вызывает изменение атрибутов, что затрагивает Layout и может вызвать уменьшение производительности. По этой причине нельзя использовать большое количество объектов или точек в svg-графике.

В то же время Adobe After Effects — самый простой и удобный инструмент для работы с Morphing форм. В Adobe Animate этот процесс может вызвать недоумение: в некоторых случаях форма может просто исчезнуть в промежуточных кадрах. В Adobe After Effects такого не происходит, и это позволяет делать пластичную и приятную анимацию.

В Adobe After Effects можно работать со слоями в режиме 3D. Там есть объект камеры в двух режимах: 1-node и 2-node Camera. А также «пустые» Null-объекты вроде Group без ассоциированной графики в Three.js, которые тоже можно использовать для построения Camera Rig для управления анимацией камеры. Такую анимацию 3D нужно воспроизводить в режиме html на сайте. В этом случае анимация будет использовать 3D css-трансформации.

Анимация для специального проекта lenta.ru «Титаны в трико», VisCom by RamblerGroup.

Сайты: Adobe Animate, createjs.com
