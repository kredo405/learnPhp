Глава 3.6. Методика реализации SMIL анимации

Для чего это руководство и как им пользоваться

Анимация SMIL предназначена для векторных объектов внутри SVG.

Это руководство поможет понять, как правильно реализовать SMIL анимацию для типовой веб-страницы.

Типичный процесс реализации анимации:

  1. Получаем вёрстку со статическими SVG-объектами.
  2. Группируем объекты в svg, которые будем анимировать.
  3. Добавляем анимацию в объекты.
  4. Разбиваем дублирующую анимацию на отдельные svg-файлы и делаем ссылки на них в HTML.

Алгоритм создания SMIL анимации

Рассмотрим реализацию анимации на примере. Согласно ТЗ, самолёт должен по синусоиде пролетать по всей карте. Нужно сделать покачивания самолёта, взмахивание баннера и исчезновение/появление звёзд.

Шаг 1. Группируем объекты

В начальном виде svg состоит из набора path, rect, circle и т. д. Нужно сгруппировать объекты, которые мы будем анимировать, согласно анимации. А именно: всё, связанное с баннером — отнести в группу баннера, всё, что относится к телу самолёта — разместить отдельно.

Шаг 2. Интерпретируем анимацию на изменение параметров объектов

Очень часто дизайнеры дают «человеческое» описание анимации, физике или природе трансформаций объектов. Программистам же, в свою очередь, нужно понять, какие параметры стоит изменять, чтобы получить такое событие. Рассмотрим примеры из нашего ТЗ:

  • «покачивание самолёта» означает увеличение/уменьшение объекта по оси Y (scaleY) и скручивание по оси X (skewX). Такие изменения ещё называют «псевдо 3D-вращение»;

  • «взмах баннера» — изменение угла поворота вокруг опорной точки в месте крепления баннера к самолёту;

  • исчезновение/появление звёзд — моментальная смена прозрачности от 0 до 1.

Шаг 6. Группируем анимацию по файлам

Одно из преимуществ SMIL анимации — возможность описать всю логику процесса внутри svg-файла, а в html только лишь вставить ссылки на этот файл в тег img: не нужно дублировать много кода внутри html-страницы. А ещё с помощью CSS стилей можно расположить объекты по всей карте.

До:

index.html

<div class="map__decor">
  <img src="img/decor.svg" alt="">
</div>

map.scss

.map__decor {
  img {
    display: block;
    width: 100%;
    height: auto;
  }
}

Итог

Следуя алгоритму создания SMIL анимации, мы идём от общего к частному: сначала группируем объекты в крупные блоки, затем анимируем каждую деталь внутри. Создавая анимацию одного объекта внутри html-страницы, мы можем заменить его тегом img со ссылкой на svg, продублировать на странице и с помощью стилей расположить в нужном месте.
