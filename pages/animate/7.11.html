Базовая теория

Глава 7.11. Конструкции Camera Rig: 1-node, 2-nodes и 1-node с "нулём"

Конструкции Rig для камеры состоят из набора групп Group, которые объединяются в одну систему и в финальный узел которой помещается камера.

Group в некоторых источниках и программах называются «нулём» (Null или Null Object), поскольку не содержат ассоциированной с ними графики.

Работа с Camera Rig мало отличается от работы со структурными Rig’ами, с которыми мы познакомились в главе 7.3. Методика структурного Rigging

Все конструкции Rig созданы для реализации конкретного поведения камеры. Поэтому они позволяют получать предсказуемый результат и лишены тех сложностей в определении углового положения во время анимации — «странных движений», о которых мы говорили ранее.

Рассмотрим сначала простые конструкции Camera Rig, которые состоят из 2-3 элементов.

1-node Camera Rig

Конструкция 1-node Camera Rig предназначена для прямых проездов при зафиксированном направлении камеры. Это заранее построенные рельсы, по которым перемещается камера.

Такое перемещение камеры называется «Проезд» или «Truck». Используется для перемещения вдоль ландшафта или реже для перемещения между объектами по линии.

Отличие 1-node Camera Rig от анимации камеры по прямой в мировой СК в том, что Rig — это жестка конструкция, которую саму вы можете перемещать, ставить под углом и т. п. При этом вам не нужно менять формулы для расчётов. В СК узла Rig камера как перемещалась вдоль оси Xrig, так и продолжает перемещаться вдоль этой оси.

2-node Camera Rig

Это то же, что и 1-node Camera Rig, только добавляется дополнительный узел — node — «Цель» или Target. Как и в случае с 2-node Camera, реализовать поворот к цели можно с помощью метода lookAt.

2-node Camera Rig часто используется для «представления» объекта: пролета мимо объекта с фокусом на нём. При этом для придания значимости объекту или создания пафосного эффекта объект часто снимают снизу вверх с широким углом fov камеры.

2-node Camera Rig также можно использовать для анимации реакции камеры на движение курсора по экрану. Это позволяет не только поворачиваться сцену вокруг одной оси, но и удаляться от центра сцены при удалении курсора от центра экрана.

1-node Camera Rig с «нулём»

Предыдущие конструкции использовались для прямолинейных проездов. 1-node Camera Rig с «нулём» создан для облётов вокруг объекта на одной высоте.

1-node Camera Rig с «нулём» позволяет устанавливать плоскость облета под углом и в процессе анимации менять расстояние до объекта — радиус.

Рассмотрим на примере 1-node Camera Rig с «нулём» реализации по схеме, которую изучили ранее:

Подготовка описания или схемы устройства

П 1. Мы нарисовали схему.

ПП 2-3. Нам понадобится

  • группа в основе, которую мы сможем перемещать и устанавливать направление — Root,

    В качестве Root может выступать сам объект Rig, если мы программируем Rig в виде класса, который наследуется от Group.

  • группа, которую мы будем вращать вокруг собственной оси Y — Null,

  • камера, которую мы будем перемещать вдоль одной оси, в данном случае удобнее выбрать ось Z, так как камера по умолчанию направлена вдоль Z, и нам не будет необходимости поворачивать её дополнительно.

    В качестве последнего узла мы можем использовать ещё одну группу Camera Null. С ней мы будем проводить манипуляции, а камеру без изменений поместим внутрь Camera Null или при рендеринге возьмем матрицу трансформаций в мировой СК и применим к камере или любому другому объекту. Например, эту же матрицу трансформации можно применить к группе источников света.

П 4. Соединим так: Root -> Null -> Camera

П 5. Ограничения:

  • Root без ограничений (здесь и далее будем считать, что можем размещать Rig через его Root в любое необходимо положение и помещать его в любой родительский элемент, если движение нужно реализовывать в его СК).
  • Null меняем только rotation.y в неограниченных пределах
  • Camera Null меняем только position.z теоретически от 0 до бесконечности, на практике мы можем чётко определить границы значения, например, можем камера может подъезжать не ближе, чем на 500 отн. ед. и отъезжать не дальше 2000 отн. ед.

Сборка конструкции в коде

class CamRig extends Group {

  #angle
  #minRadius
  #maxRadius
  #radius

  constructor() {
    this.super();

    // Описываем внутренние параметры и их ограничения
    this.#angle = 0;
    this.#minRadius = 500;
    this.#maxRadius = 2000;
    this.#radius = this.#minRadius;

    this.constructRigElements();
  }
 
  constructRigElements {
    // Создаем необходимое количество групп
    // Root - сам класс CamRig
    const null = new THREE.Group();
    const camNull = new THREE.Group();

    // Соединяем в конструкцию
    null.add(camNull);
    this.add(null);

    // Размещаем узлы в начальное положение
    camNull.position.z = params.#radius;    
  }

  // Пишем метод, который на основании текущих значений параметров обновляет состояние конструкции
  invalidate() {
    this.null.rotation.y = this.#angle;
    this.camNull.position.z = params.#radius;    
  }
}

Описываем внешние параметры и методы

class CamRig extends Group {

#angle #angleChanged #minRadius #maxRadius #radius #radiusChanged

constructor () { this.super ();

this.#angle = 0; this.#minRadius = 500; this.#maxRadius = 2000; this.#radius = this.#minRadius;

this.#angleChanged = false; this.#radiusChanged = false;

this.constructRigElements (); }

// Добавляем setter и getter для внутренних параметров // Устанавливаем флаги set angle (value) { if (this.#angle === value) return; this.#angle = value; this.#angleChanged = true; }

get angle () { return this.#angle; }

// Учитываем ограничения set radius () { if (this.#radius === value || value > this.#maxRadius || value < this.#minRadius) return; this.#radius = value; this.#radiusChanged = true; }

get radius () { return this.#radius; }

get maxRadius () { return this.#maxRadius; }

get minRadius () { return this.#minRadius; }

constructRigElements { ...
}

// Добавляем проверки флагов invalidate () { if (this.#angleChanged) { this.null.rotation.y = this._angle;

// Сбрасываем флаг this.#angleChanged = false; }

if (this.#radiusChanged) { this.camNull.position.z = params._radius;

// Сбрасываем флаг this.#radiusChanged = false; }
} }

После этого вовне анимируем только angle и radius. А maxRadius и minRadius нужны для того, чтобы при написании Tween не выйти за границы, иначе easing оборвется на границе, и камера резко остановится.
