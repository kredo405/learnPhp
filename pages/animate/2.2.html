Базовая теория

Глава 2.2. Отслеживания изменений в CSS. Процесс перерисовки страницы.

Как говорилось выше, линейная анимация — это процесс плавной смены свойств CSS. Тем не менее за каждым таким изменением стоит большая работа на стороне браузера. В этой главе рассмотрим, какие процессы происходят под капотом браузера, когда меняется то или иное свойство в CSS.

Перерисовка страницы называется рендерингом. Рендеринг — это заполнение страницы пикселями, полученными в результате выполнения HTML, CSS и JS кода. Весь процесс рендеринга разбит на области, которые выполняются последовательно одна за другой.

Есть пять областей, на которые может повлиять разработчик веб-приложений. При изменениях в какой-либо из них происходит перерисовка следующих за ней областей (на изображении слева направо).

  • JavaScript вызывается при изменениях в DOM, например: изменение контента в элементах, добавление DOM-элементов, изменение атрибутов.

  • Пересчёт стилей — область расчёта каскада стилей и определения финальных стилей элемента. Вызывается при добавлении новых блоков CSS.

  • Расстановка определяет размеры и позиции всех элементов. Эта перерисовка запускается при изменении свойств, которые влияют на размеры, отступы и позиции элементов, например: width, height, margin, max-width, padding, left и т. д. В некоторых источниках этот процесс называется reflow.

  • Рисование — область, отвечающая за внешний вид элементов. Триггером для её запуска является изменение свойств, связанных с внешним видом элемента, например: background, color и т. д.

  • Компоновка — последняя часть процесса рендеринга страницы. На этом этапе определяется порядок наложения слоёв, создаётся прозрачность и накладывается трансформация. Применялись следующие свойства: z-index, transform, opacity, filter.

Если нужно получить плавную анимацию, процесс повторного рендеринга страницы должен занимать не более 16.667 мс (60 раз/сек). Самый простой способ оптимизировать отрисовку — использовать анимирование свойств, которые применяются в областях «рисование» и «компоновка».

Пример такой оптимизации — замена свойства left на transform: translateX:

До:

.block {
  position: absolute;
  transition: left 0.3s ease;
  left: -300px;
};
.block:hover{
  left: 0;
}
